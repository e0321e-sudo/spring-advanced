# 🚀 [Spring 3기] CH 3 심화 Spring_코드 개선 과제
기존 코드의 버그를 수정하고, 아키텍처를 개선하며, 성능을 최적화하는 과정을 기록했습니다.

---

## 🛠 과제별 해결 과정 (Step-by-Step)

### **Lv 0. 초기 설정 및 환경 구축**
* **내용**: 프로젝트 가동을 위한 인메모리 데이터베이스(H2) 설정 및 JWT 보안 키 세팅.
* **해결 과정**:
    1. `build.gradle`에 프로젝트 필수 의존성을 구성하고, 가벼운 테스트 환경을 위해 H2 데이터베이스 연결 설정을 수행함.
    2. 서버 보안을 위해 JWT(`jwt.secret.key`)와 토큰 만료 시간 설정을 `application.yml`에 체계적으로 관리하도록 구축함.
* **성과**: 복잡한 외부 설정 없이도 즉시 개발과 테스트가 가능한 독립적인 백엔드 환경을 완성함.

### **Lv 1. Argument Resolver를 활용한 인증 자동화**
* **내용**: `@Auth` 어노테이션과 `AuthUserArgumentResolver` 구현 및 `WebConfig` 등록.
* **성과**: 컨트롤러 메서드 파라미터에서 로그인한 유저 정보(`AuthUser`)를 즉시 주입받도록 자동화함. 반복적인 토큰 파싱 코드를 제거하여 비즈니스 로직의 응집도를 높임.

### **Lv 2. 코드 개선 및 가독성 확보**
* **Early Return 적용**: `AuthService.signup`에서 이메일 중복 체크를 로직 최상단으로 이동. 불필요한 암호화(Encoding) 연산을 방지하여 서버 자원을 효율화함.
* **Refactoring**: `WeatherClient` 등에서 불필요한 `else` 문을 제거하여 코드의 깊이(Depth)를 줄이고, 읽기 좋은 평탄한 코드로 개선함.
* **Validation 통합**: 서비스 레이어의 복잡한 `if` 검증 로직을 DTO의 `@Valid` 어노테이션으로 이관하여 유지보수성을 향상함.

### **Lv 3. N+1 문제 해결 및 성능 최적화**
* **문제**: Todo 목록 조회 시 연관된 User 정보를 가져오기 위해 다수의 추가 쿼리가 발생하는 N+1 문제 확인.
* **해결**: `TodoRepository`에서 JPQL `fetch join` 대신 **`@EntityGraph`**를 적용. 단 한 번의 JOIN 쿼리로 데이터를 조회하도록 개선하여 데이터베이스 부하를 최소화함.

### **Lv 4. 테스트 코드 신뢰성 확보 및 버그 수정**
* **PasswordEncoder**: `matches()` 메서드의 인자 순서 오류를 바로잡아 인증 로직 검증을 정상화함.
* **Exception Sync**: 테스트가 기대하는 예외 타입과 서비스 예외 타입을 일치시켜 테스트 신뢰도를 확보함.
* **NPE 방어**: `ManagerService`에서 발생할 수 있는 `NullPointerException`을 조건문 우선순위 조정을 통해 안정적으로 해결함.

📝 회고 (Retrospective)
기존 프로젝트를 분석하며 Filter, Resolver, @EntityGraph를 통해 데이터 흐름과 성능을 최적화하고 리팩토링의 실무적 가치를 확인했습니다.
특히 NPE 방지를 위한 조건문 우선순위 설계나 테스트 케이스와 실제 예외 타입 간의 불일치를 해결하며, 견고한 시스템 구축을 위한 방어적 프로그래밍과 디테일한 디버깅의 중요성을 체감했습니다.
